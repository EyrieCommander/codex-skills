# Voice Calibration

Use this reference when the user gives a voice sample, names a target writer or brand, or asks for a stronger point of view.

## General Method

Treat voice as decision-making, not costume.

Extract:

- rhythm: sentence length, paragraph length, abruptness, willingness to ramble;
- stance: certainty, doubt, irony, moral pressure, curiosity;
- vocabulary: plain words vs technical terms, favorite abstractions, tolerance for slang;
- syntax: fragments, questions, parentheticals, lists, parallelism;
- evidence style: anecdote, data, quotation, intuition, source links;
- social posture: warm, adversarial, confessional, didactic, exploratory.

Do not mimic surface tics too hard. A good voice match preserves posture and cadence without turning into parody.

## When No Sample Is Provided

If the user asks for a specific person's voice and provides no sample, ask for a sample unless the task is low-stakes and the requested traits are already clear. Do not infer a private person's voice from local memory, private correspondence, meeting notes, or unpublished writing.

For public figures, brands, or publications, use only public source material the user provides or that you can verify in the current task. Keep the result closer to posture and rhythm than surface imitation.

## Voice-Sample Workflow

1. Read the sample for rhythm and stance before touching the target text.
2. Identify three to five traits worth preserving.
3. Rewrite the target text using those traits.
4. Remove parody: repeated catchphrases, exaggerated punctuation, and forced quirks.
5. Run the integrity pass from `ai-writing-tells.md`.

## Boundary Rule

Do not copy private writing samples into public artifacts. If the source is private, describe the voice traits abstractly and keep the sample in the owning workspace.
